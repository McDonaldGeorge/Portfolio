Project Overview

This project explores the use of machine learning to build a short-term weather prediction system. Using historical hourly weather data from Kaggle, I developed a regression pipeline to forecast temperature up to 30 days beyond the end of the dataset.

The system is built around Ridge Regression with cross-validation, focusing on predicting future temperatures for the city of Portland based on meteorological variables such as humidity, pressure, wind speed, and wind direction. The work includes data cleaning, feature engineering, model training, evaluation, and using the final model to extrapolate future temperature trends.

This is a completed project, originally developed as part of my CI513 module in 2024, and it demonstrates my ability to design an end-to-end regression pipeline on a large real-world dataset.

Dataset

The project uses the Historical Hourly Weather Data 2012–2017 dataset from Kaggle, which contains meteorological readings for 36 cities around the world.

  • Source: Kaggle – Historical Hourly Weather Data (2012–2017)
  • Original scope: 36 cities × ~45,254 hourly entries per city
  • Variables: Temperature, humidity, pressure, wind speed, wind direction, etc.

For this project, I focused on Portland, extracting its data from the collection of CSV files and merging them into a single cleaned DataFrame. This allowed me to design and evaluate a weather model specific to one location while still leveraging a large historical dataset.

Data Cleaning & Transformation

A major part of the work involved transforming the raw Kaggle data into a form suitable for modelling. This included:

  • Merging multiple city-level CSV files into a single DataFrame for Portland.
  • Converting temperature from Kelvin to Celsius for readability and interpretability.
  • Parsing the timestamp column and creating derived features such as Year, Month, Day, and Hour.
  • Handling missing values using dropna() for incomplete rows.
  • Removing duplicate records via drop_duplicates() to avoid skewing the model.
  • Retaining genuine zeros (e.g. calm winds) to preserve valid meteorological readings.

After cleaning and transforming the data, I exported the processed dataset to a new CSV file specifically for modelling, ensuring a reproducible pipeline and separation between raw and prepared data.

Model & Approach

The core of the system is a Ridge Regression model, implemented with RidgeCV from scikit-learn to automatically select the optimal regularisation parameter (λ). Ridge Regression was chosen due to the likelihood of multicollinearity between weather features such as humidity, pressure, and temperature.

Key steps in the modelling process:

  • Feature Selection: Input features included humidity, pressure, wind speed, wind direction, and other meteorological readings. Date-related columns were transformed and indexed primarily for ordering rather than direct input.
  • Cross-Validation: I used Repeated K-Fold Cross-Validation (10 splits, repeated 5 times, fixed random state) to obtain a more stable estimate of model performance and a robust λ selection.
  • Training: The cleaned dataset was split into training and test sets, and the Ridge model was fitted on the training data, with evaluation performed on the held-out test set.

Evaluation & Results

The model was evaluated using error-based metrics, particularly:

  • Mean Absolute Error (MAE)
  • Mean Squared Error (MSE)

The final Ridge model achieved an MAE of approximately 3.98°C, meaning that on average the temperature predictions were within about 4 degrees of the actual values. While not perfect, this represents a reasonably accurate baseline given the complexity and variability of weather.

I also generated:

  • A scatter plot comparing predicted vs actual temperatures.
  • Residual plots to visualise where the model performed well and where errors were larger.

These visualisations showed a dense cluster of correctly predicted values around the ideal line, with some spread indicating outliers and the inherent difficulty of capturing all patterns in a simple linear model.

Future Predictions

One of the main goals of the project was not only to fit a model to historical data, but to use it to generate predictions 30 days into the future.

To do this:

  • I extended the dataset by creating 744 new hourly rows (30 days × 24 hours + remaining hours).
  • For each new timestamp, I filled non-target features using historical averages for matching dates across years.
  • The trained Ridge model was then used to predict future temperatures for these new rows.

Finally, I plotted these predictions to visualise how the model expects temperature to evolve beyond the end of the original dataset, giving a continuous forecast window.

Reflections & Future Work

Although the system is functional and capable of producing reasonable short-term forecasts, there is still room for improvement.

  • Experimenting with alternative models such as Lasso, Elastic Net, or tree-based methods.
  • Exploring time-series-specific models like ARIMA or more advanced deep learning architectures.
  • Incorporating additional years or external weather sources to enrich the dataset.
  • Using metrics such as RMSE or Median Absolute Error for more nuanced evaluation.

This project gave me hands-on experience working with large datasets, cleaning and transforming time series, applying regularised regression, and critically evaluating model performance in a realistic forecasting context.
